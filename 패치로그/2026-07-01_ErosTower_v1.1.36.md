# Eros Tower v1.1.36

## 콜드스타트 흡수 방식 재정렬
- `coldStartMaxChunksPerRun` 기본값을 `256`으로 올려 한 번의 콜드스타트 실행에서 가능한 많은 원천 청크를 처리 대상으로 잡는다.
- 이 값은 내부 로어 unit 개수 제한이 아니라 소스 청크 후보량이다. 한 청크에서 생성되는 `psycheUnits` 개수에는 별도 hard cap을 두지 않는다.
- 1.1.35에서 저장값이 `3` 또는 과거 소량 기본값으로 남아 있으면 1.1.36 기본값으로 자동 승격한다.

## 최소 호출 배치 처리
- 소스 청크마다 API를 따로 호출하지 않고, 가능한 큰 배치로 묶어 한 요청에 여러 청크를 넣는다.
- run toast/run log에 `processed chunks / calls / units / errors`를 표시해 실제 처리량 대비 호출 수를 확인할 수 있게 했다.
- 소스 ingest 출력 토큰을 크게 확보해 한 호출에서 많은 내부 unit을 반환할 수 있게 했다.

## 내부 로어 분해 유지
- 한 원천 청크에 인물 수십~수백 명이 들어 있어도 통합 요약 하나로 저장하지 않고, 가능한 만큼 별도 unit으로 분해하도록 지시한다.
- dense/long 로어는 모델이 `complete:true`를 명시해야 완료로 본다.
- `complete:false` 또는 부분 추출이면 같은 콜드스타트 실행 안에서 이미 추출한 `previousUnits`를 제공하고 이어 추출한다.
- 이어 추출 시 중복 unit은 청크 해시와 unit 내용 해시 기반 ID로 덮어써져 중복 누적을 줄인다.
