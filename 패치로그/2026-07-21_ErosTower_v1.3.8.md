# Eros Tower 1.3.8

## 버전

- 정식 업데이트 버전: `1.3.8`
- 호환 업데이트 채널 버전: `4.0.45`
- 기준 파일: `ErosTower.v1.update.js`

## 증상

- 1.3.7에서 잘린 Psyche JSON을 자동 재시도하도록 수정했지만, 큰 인물 시트의 표시 언어 보정이 다시 출력 한도에 걸릴 수 있었습니다.
- 대표 오류는 `Psyche 인물 스테이터스 보정 실패`와 `...한국어로 준비하지 못했습니다: status`였습니다.

## 원인

- `status` 하나만 잘못된 언어여도 보정 호출에 전체 인물 초기화 프롬프트와 전체 상태·검색 컨텍스트·최근 대화를 다시 넣었습니다.
- 전체 초기화 프롬프트는 모든 능력·아이템을 출력하라고 지시하면서, 뒤의 보정 문구는 잘못된 필드만 출력하라고 지시해 계약이 충돌했습니다.
- 1.3.7의 JSON 복구 요청도 원래 요청에 필요한 모든 근거 사실을 유지하라고 했기 때문에, 전용 보정 요청에서는 전체 시트를 다시 출력하려는 경향이 남았습니다.
- 초기 인물 바인딩이 넣는 알려진 시스템 상태 `active`도 플레이어 표시 문구 검사에 걸려 불필요한 모델 보정을 일으켰습니다.
- 정규 인물 근거 병합이 부분 델타를 완전한 인물 객체로 정규화할 때, 모델이 보내지 않은 컬렉션에 빈 기본값을 만들 수 있어 기존 조건 목록을 덮을 가능성이 있었습니다.

## 수정

- 전체 인물 초기화와 표시 언어 보정을 서로 다른 요청 계약으로 분리했습니다.
- 표시 언어 보정은 검색 컨텍스트, 전체 상태 스냅샷, 최근 대화를 다시 싣지 않습니다. 선택 인물 식별자, 출력 언어, 실패 경로, 해당 현재 값만 담은 소형 payload를 사용합니다.
- `status` 문제는 `id`, `name`, `status`만 반환하게 하며, 능력·아이템 문제도 해당 ID와 지정된 표시 필드만 반환하게 합니다.
- 보정 JSON이 잘려 한 번 재시도할 때도 전체 시트를 반복하지 못하도록 전용 최소 델타 지시를 유지합니다.
- `active`, `ready`, `dead`, `incapacitated` 등 알려진 시스템 상태는 한국어·영어·일본어·중국어의 안전한 고정 매핑으로 먼저 현지화합니다. 임의의 서술 상태는 추측 번역하지 않고 Psyche 보정 대상으로 남깁니다.
- 조건 언어 검사는 canonical `name`보다 `displayName`을 우선해, 이미 올바른 현지화가 있어도 실패하던 경우를 제거했습니다.
- 캐릭터 부분 커밋의 희소성을 보존해, 응답에 없던 별칭·스탯·조건·능력·아이템·근거 컬렉션이 빈 기본값으로 기존 상태를 덮지 않게 했습니다.

## 구조와 안전성

- 저장 키, 저장 스키마 버전, 캐릭터·능력·아이템 데이터 형식은 변경하지 않았습니다.
- 언어 보정 결과는 기존 `resolveEvidenceCommit → restrictGameplayBootstrapCommit → applyStateCommit` 근거 검증·선택 인물 제한·병합 경로를 그대로 통과합니다.
- 보정은 전체 객체 교체가 아니라 부분 델타 병합이며, 기존 능력 96개·아이템 96개·조건 목록·비용·수량이 유지되는 회귀 검사를 추가했습니다.
- 알려진 시스템 상태 외의 자유 서술은 로컬에서 만들거나 번역하지 않습니다.
- 1.3.6 저장소 블롭 안정화와 1.3.7 JSON 후보 복구·1회 자동 재시도는 그대로 유지합니다.

## 회귀 검사

- 능력 96개, 아이템 96개, 긴 대화 이력이 있는 인물에서 `status` 하나만 실패한 상황을 재현했습니다.
- 첫 보정 응답을 출력 한도에서 잘라 두 번째 응답으로 복구하고, 정확히 한 번만 재시도하는지 확인했습니다.
- 보정 system prompt에 전체 초기화 계약이 없고, user payload에 96번째 능력·아이템 및 최근 대화가 포함되지 않는지 확인했습니다.
- 재시도 지시가 전체 인물 시트 재출력을 금지하는지 확인했습니다.
- 부분 `status` 델타 적용 뒤 능력 96개, 아이템 96개, 조건 2개, 능력 비용과 아이템 수량이 모두 보존되는지 확인했습니다.
- `active → 활성`은 모델 호출 없이 처리되고, 임의 상태 `Watchful`은 로컬 추측 번역 없이 보정 대상으로 남는지 확인했습니다.
- 조건의 한국어 `displayName`이 canonical 영문 `name` 때문에 실패하지 않는지 확인했습니다.
