# Eros Tower 1.2.7

## 기준

- `1.2.7-game-agent-test.47` 최종 기준선에서 저장 구조를 개선한 `.48` 최종테스트본을 정식판으로 승격했다.
- 구조, 시스템, 기억, 상태관리의 의미 데이터는 삭제하거나 요약하지 않는다.
- 기존 정식판의 `eros_tower_v02:` namespace와 `eros_tower_v01:` 호환 읽기를 유지한다.

## 저장 구조 v3

- 현재 상태, 백업, 최대 16개 스냅샷의 큰 의미 상태를 내용 해시 기반 불변 청크로 저장해 동일한 장기기억·비밀·로어·관계·인물·세계 상태의 중복 사본을 제거했다.
- 각 청크, 섹션, 전체 상태의 체크섬을 읽을 때 검증하며, 현재 상태·백업·스냅샷의 모든 참조를 확인한 뒤에만 미사용 청크를 정리한다.
- 다국어 UTF-8을 지원하는 무손실 LZW v2 압축을 추가하고 raw, 기존 dict-v1, LZW v2 중 실제 저장 바이트가 가장 작은 형식을 선택한다.
- 기존 raw와 dict-v1 상태는 그대로 읽고 첫 로드부터 v3로 무손실 이관한다. 스냅샷은 부하를 줄이기 위해 지연 이관한다.
- 독립 복구 payload를 백업에 유지해 공유 청크 손상이나 누락 시에도 복구할 수 있다.

## 저장소 수명 관리

- Canonical 본문을 전역 내용 주소 청크로 공유해 채팅별 중복 저장을 줄였다.
- 통신 스레드를 100개 메시지 페이지로 나눠 변경되지 않은 과거 페이지의 재저장을 피한다.
- 임베딩 캐시 namespace를 최근 4개로 제한한다.
- 현재 범위 리셋 시 상태·백업·스냅샷·Run Log·Canonical·통신 관련 데이터를 함께 제거하며, 권위 있는 채팅 목록이 있을 때만 삭제된 채팅의 고아 저장 범위를 정리한다.
- 대시보드에 플러그인 전체·현재 채팅·카테고리별 실제 UTF-8 저장 바이트와 키 수를 표시한다.

## 기본 설정

- 에로스 타워 본체는 설치 후 활성 상태를 유지한다.
- 관리형 프롬프트 캐싱은 기본 `OFF`다.
- 게임플레이 조언자 `으아악씨`는 기본 `OFF`다.
- 통신 상주자 `모시모씨`는 기본 `OFF`다.
- 기존 사용자가 해당 기능을 명시적으로 켜둔 설정은 마이그레이션에서 보존한다.
- 저장 구조 v3의 무손실 압축은 저장량 감소의 핵심이므로 활성 상태를 유지한다. 이는 위의 선택형 프롬프트 캐싱과 별개다.
