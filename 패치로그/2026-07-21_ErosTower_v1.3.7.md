# Eros Tower 1.3.7

## 버전

- 정식 업데이트 버전: `1.3.7`
- 호환 업데이트 채널 버전: `4.0.44`
- 기준 파일: `ErosTower.v1.update.js`

## 증상

- 으아악씨에서 선택한 인물의 Psyche 스테이터스를 준비할 때 `Psyche 인물 준비 실패: json-parse-failed`가 반복될 수 있었습니다.
- 제공자 요청은 완료됐지만 `runStateCommit`이 모델의 공개 응답에서 상태 커밋 객체를 얻지 못한 경우였습니다.

## 원인

- 기존 전용 파서는 응답에서 첫 번째 `{...}` 후보만 검사했습니다. 앞의 설명에 `{id, name}` 같은 중괄호가 있으면 뒤의 완전한 JSON을 검사하지 않고 실패했습니다.
- `{"commit": {...}}`, `{"stateCommit": {...}}`, `{"delta": {...}}`처럼 명시적으로 감싼 응답은 외부 객체를 그대로 커밋으로 취급해 실제 델타를 적용하지 못했습니다.
- 출력 한도에서 JSON이 잘리거나 공개 JSON 없이 추론만 반환돼도 형식 재시도 없이 즉시 `json-parse-failed`를 반환했습니다.
- 인물 준비 전용 설정은 사용자가 4,096보다 큰 출력 한도를 설정해도 최대 4,096으로 다시 제한했습니다.

## 수정

- 모든 균형 잡힌 JSON 객체 후보를 순서대로 검사하고, 실제 상태 커밋 키가 있는 객체를 선택합니다.
- 명시적인 `commit`, `stateCommit`, `state_commit`, `delta`, `result` 래퍼에서 상태 커밋을 안전하게 해제합니다.
- 정상적인 빈 커밋 `{}`은 기존처럼 `empty-commit`으로 처리하며 불필요하게 재시도하지 않습니다.
- 비공개 `<analysis>`, `<think>`, `<reasoning>` 내용은 상태로 가져오지 않습니다.
- 완결된 JSON을 얻지 못했을 때만 낮은 온도·낮은 추론 수준·최소 4,096 출력 토큰으로 간결 JSON 복구 요청을 정확히 한 번 실행합니다.
- 재시도에도 실패하면 출력 한도 종료, 추론 전용 응답, 거부, 도구 호출, 일반 파싱 실패를 구분한 사용자 메시지를 반환합니다.
- 인물 준비 경로가 사용자가 설정한 출력 한도를 최대 8,192까지 존중합니다.
- 로컬 규칙으로 인물 스테이터스를 추측 생성하지 않으며, 실제 모델이 반환한 완결 JSON만 기존 근거 검증·커밋 경로에 전달합니다.

## 호환성과 안전성

- 상태 저장 스키마, 캐릭터·능력·아이템 구조, 스냅샷, 체크섬은 변경하지 않았습니다.
- 1.3.6의 장기기억 블롭 중복 방지와 제한 동시 로딩 패치를 그대로 유지합니다.
- 정상 JSON과 정상 빈 커밋은 추가 호출 없이 기존 경로로 처리됩니다.
- 거부·도구 호출은 무의미한 재시도를 하지 않습니다.

## 회귀 검사

- 앞쪽의 비 JSON 중괄호 뒤에 있는 정상 상태 JSON을 복구했습니다.
- 명시적 `commit` 래퍼를 해제해 내부 커밋을 적용했습니다.
- 닫히지 않은 비공개 분석 블록의 JSON은 상태로 가져오지 않았습니다.
- 첫 응답을 `finish_reason=length`와 잘린 JSON으로 만들고, 두 번째 간결 응답을 완전한 JSON으로 반환하는 재현에서 정확히 2회 호출 후 상태 적용에 성공했습니다.
- 재시도 요청의 낮은 온도, 4,096 이상 출력 한도, 명시적 JSON 전용 계약, `length → stop` 응답 메타데이터를 확인했습니다.
- 인물 준비 설정의 8,192 출력 토큰 보존을 확인했습니다.

## 검증

- 세 배포 파일 모두 `node --check` 통과.
- ESLint 8 정적 검사(`no-redeclare`, `no-dupe-keys`) 통과.
- Node 힙 `64MB`에서 기준 파일과 호환 파일의 전체 내장 회귀 검사 통과: `196`개 테스트, `1,633`개 boolean 검증, 오류 `0`개.
- 1.3.6 기준의 기존 false `23`개는 동일하며 신규 false는 없습니다.
- `ErosTower.v1.update.js`와 `☸에로스 타워.js`는 바이트 단위로 동일합니다.
