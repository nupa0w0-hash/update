# Eros Tower 1.3.6

## 버전

- 정식 업데이트 버전: `1.3.6`
- 호환 업데이트 채널 버전: `4.0.43`
- 기준 파일: `ErosTower.v1.update.js`

## 원인

- 장기기억 원문 해시가 같아 동기화 결과가 `unchanged`여도 `evidence.at`과 `chunk.syncedAtTurn`을 현재 시각·턴으로 다시 기록했습니다.
- 상태 저장은 장기기억 12개 단위의 내용 주소형 블롭을 사용하므로 이 두 진단용 값만 달라져도 모든 장기기억 블롭이 새 세대로 생성됐습니다.
- 매 턴 저장하는 스냅샷이 각 세대를 정상 참조했기 때문에 GC는 이를 삭제할 수 없었습니다.
- RisuAI의 플러그인 스토리지 읽기는 전체 데이터베이스 스냅샷을 만들 수 있어, 수백 개 블롭을 한꺼번에 읽는 기존 `Promise.all` 경로가 OOM 위험을 더 키웠습니다.

## 수정

- 같은 장기기억 원문 해시에는 기존 `evidence`와 `chunk.syncedAtTurn`을 보존합니다.
- 원문·범위·추출 상태 등 실제 의미가 달라질 때는 기존처럼 새 블롭과 체크섬을 생성합니다.
- 상태 블롭 복원 동시성을 무제한에서 최대 2개로 제한했습니다. 복원 순서와 배열·객체 조립 순서는 그대로 유지합니다.
- 저장 스키마 버전, 키 형식, 블롭 형식, 스냅샷 형식은 변경하지 않았습니다.

## 모시모씨 작성창 후속 수정

- 1.3.5의 `CALL_INSTANCE` 프록시 차단, 실제 답장 전용 `. → .. → ...` 모션, 무응답·무시·지연·거부·연결 불가 상태, 세계관별 통신 용어와 디자인, 연락처 뒤로가기와 작은 이름 검색 버튼이 1.3.6 본문에 모두 승계됐음을 확인했습니다.
- Risu SafeDOM 안의 `contenteditable` 직접 편집에 의존하던 작성창을 제거했습니다. 메시지 작성 영역을 누르면 Risu에서 실제 입력이 가능한 자유 입력 창을 열고, 입력한 초안을 패널에 미리 보여준 뒤 작은 전송 버튼으로 보냅니다.
- 초안이 없는 상태에서 전송·말하기 버튼을 누르면 입력 창부터 열어 빈 메시지 오류로 끝나지 않게 했습니다.
- `연결 불가`, `거절됨`, `부재중` 같은 종료 결과와 수신 대기 통화는 메시지 작성 영역과 전송 버튼을 잠그지 않습니다. 통화 결과 표시와 메시지 작성 가능 상태를 분리했습니다.
- 메시지 초안은 SafeDOM 요소의 `value`나 `innerText`를 읽지 않고 런타임 초안 저장소에서만 가져오며, 기존 `CALL_INSTANCE` 프록시 서명 차단을 그대로 적용합니다.

## 기존 데이터와 GC

- 기존 상태·백업·스냅샷은 그대로 읽고 복원할 수 있습니다.
- 이미 누적된 과거 블롭은 살아 있는 스냅샷을 임의로 삭제하지 않습니다.
- 새 버전에서 동일 블롭을 재사용하기 시작한 뒤 스냅샷 링이 교체되면, 더 이상 참조되지 않는 과거 세대를 기존 mark-and-sweep GC가 제거합니다.

## 회귀 검사

- `3,604`개 메시지, 컨텍스트 윈도우 `4`, 청크 크기 `4`로 장기기억 `900`개를 만들고 `10`턴 동안 상태·스냅샷 저장과 GC를 반복하는 검사를 추가했습니다.
- 2~10턴의 동기화는 매번 장기기억 `900`개를 모두 `unchanged`로 판정하고 첫 턴의 블롭 해시를 전부 재사용했습니다.
- 장기기억 상태 블롭은 `750 → 75`개로 `90%` 감소했습니다.
- 해당 scope 전체 키는 `762 → 87`개로 약 `88.6%` 감소했습니다.
- 10개 스냅샷을 유지한 상태에서 가장 오래된 스냅샷과 최신 스냅샷 모두 장기기억 `900`개를 복원했습니다.
- 무변경 동기화가 기존 추출 상태, 추출 오류, 증거 시각, 최초 동기화 턴을 보존하는 검사를 추가했습니다.

- 세 배포 파일 모두 `node --check` 통과.
- ESLint 8 정적 검사(`no-redeclare`, `no-dupe-keys`) 통과.
