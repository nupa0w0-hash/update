# Eros Tower 1.3.2

- 작업일: 2026-07-20
- 정식 업데이트 버전: `1.3.2`
- 호환 업데이트 채널 버전: `4.0.40`

## 문제

- 스킬·아이템 목록이 이름과 사용 가능 여부를 함께 표시하는 넓은 가로형 타일이라 한 줄에 약 2개만 보였습니다.
- 화면용 목록 생성 단계에서 종류별 30개, 전체 행동 목록 80개의 절단 상한이 적용되어 큰 스킬 카탈로그의 뒤쪽 항목이 UI에 도달하지 못했습니다.
- 절단 상한을 제거한 뒤 기존 데이터의 `무기`, `보조무기`, `공격`, `방어` 같은 세부 분류가 각각 별도 탭으로 노출되어 목록이 중구난방으로 보였습니다.
- 대시보드를 열 때마다 레거시 플러그인 인자 약 40개, 전체 데이터베이스와 로어, 현재 채팅 상태, 복구 기록, 스냅샷, 사용량, Run Log와 플러그인 저장소 전체 키·값을 한꺼번에 확인했습니다.
- 초기 화면에 필요하지 않은 상태 정합성 보정, 메모리 재색인, 마이그레이션과 저장까지 실행되어 저장량이 많을수록 첫 화면이 늦어지고, 호스트 호출 하나가 응답하지 않으면 로딩 화면이 끝나지 않을 수 있었습니다.

## 변경

- 스킬·아이템을 한 칸에 종류별 이모지와 한 줄 이름만 표시하는 인벤토리형 그리드로 변경했습니다.
- 긴 이름은 칸 너비에 맞춰 말줄임표로 처리하고 전체 이름은 버튼 설명에서 확인할 수 있게 유지했습니다.
- 사용할 수 없는 항목은 넓은 상태 문구 대신 작은 자물쇠 배지만 표시합니다.
- 칸을 누르면 기존의 읽기 전용 상세 카드에서 설명, 효과, 비용, 요구 조건, 보유량과 사용 가능 사유를 확인할 수 있습니다.
- PC와 모바일 모두 한 줄에 정확히 8칸, 최소 높이 32px로 고정했습니다. 이모지는 13px, 이름은 7px 한 줄로 표시하며 모바일에서 칸을 다시 키우는 예외를 제거했습니다.
- 한 페이지는 16개로 고정해 가득 찬 페이지가 항상 `8개 × 2줄`이 되며, 항목이 더 많으면 이전/페이지 번호/다음 버튼으로 이동합니다. 페이지 번호는 최대 5개씩 현재 페이지 주변을 보여 줍니다.
- 모든 능력 분류를 하나의 `스킬` 탭으로, 모든 소지품 분류를 하나의 `인벤토리` 탭으로 통합했습니다. `마법`, `체술`, `무기`, `보조무기`, `공격`, `방어` 같은 원래 분류 값은 삭제하지 않고 항목 상세 정보에만 표시합니다.
- 종류별 30개 및 전체 80개 UI 절단을 제거해 Psyche 상태에 저장된 스킬·아이템 전체가 분류 탭과 페이지에 포함되게 했습니다.
- 한국어·영어·일본어·중국어 페이지 이동 접근성 문구를 추가했습니다.
- 대시보드 첫 진입은 저장된 설정 한 건과 현재 캐릭터·채팅 식별 정보만 읽도록 변경했습니다. 레거시 인자 전체 조회, 전체 DB·로어 조립, 상태 보정·저장·마이그레이션은 첫 진입에서 실행하지 않습니다.
- API·에이전트·프롬프트·앨범 외 화면은 메뉴에서 선택할 때만 렌더링하고, 참고 자료·현재 채팅 상태·사용량·Run Log도 해당 탭을 처음 열 때만 읽습니다.
- 관리상태 탭도 현재 채팅의 상태·스냅샷·백업만 읽으며, 전체 DB·로어는 실제로 필요한 참고 자료 탭에서만 조회합니다.
- 전체 저장소 키와 실제 UTF-8 바이트를 순회하는 진단은 자동 실행에서 제거하고 `저장소 사용량 측정` 버튼을 눌렀을 때만 실행하도록 바꿨습니다.
- 초기 호스트 호출에 제한 시간을 두고, 로딩 화면의 `다시 확인` 버튼이 실제로 현재 채팅 확인을 재시도하도록 연결했습니다.
- 평상시 대시보드 렌더링에서는 대형 개발용 디버그 테스트 API를 만들지 않고 디버그 로그가 켜진 경우에만 설치합니다.
