# Eros Tower v1.1.34

## 핵심 변경
- 로어북 원본을 내부화할 때 항목 수가 많아 뒤쪽 로어가 빠지던 제한을 제거했다.
- `globalLore`, `localLore`, `moduleLore`, 참조 캐릭터/모듈 로어를 48개까지만 수집하던 제한을 제거했다.
- canonical source 전체를 260개까지만 유지하던 제한과 개별 로어 내용 12000자 절단을 제거했다.

## Deep Source Ingest
- 원천 자료 청크를 6000자에서 16000자로 늘리고, 경계 오버랩을 360자에서 1200자로 늘렸다.
- 실제 ingest 입력 상한을 7600자에서 18000자로 늘렸다.
- 한 번에 처리하는 내부화 청크 기본값을 16으로 올리고, 설정 상한을 64까지 열었다.
- 기존 저장값이 과거 기본값인 2~4청크로 남아 있으면 1.1.34 기본값 16으로 자동 승격한다.
- 로어북 항목을 하나의 요약으로 뭉개지 말고 인물, 세력, 장소, 규칙, 비밀, 관계, 시간선, 자원, 시스템, 지식 경계별 내부 유닛으로 분해하도록 지시를 강화했다.

## 자료망 보존
- 소스/청크/내부 유닛에 `sourceClass`, `loreStability`, `retention`을 상속해 불변/상시/선택/문맥 로어 성격이 내부화 중 사라지지 않게 했다.
- canonical lore ledger가 많은 로어에서 뒤쪽 항목을 밀어내지 않도록 정리했다.
- 내부 association graph 기본 엣지 수를 2400으로 늘리고, 노드 상한도 설정 규모에 맞춰 더 넓게 잡았다.
- 기존 저장값이 과거 graph 기본값 800이면 2400으로 자동 승격한다.
- run log/토스트에서 소스 청크 흡수 완료율과 미완료 소스 수를 확인할 수 있게 했다.
