# Eros Tower 1.3.4

## 버전

- 정식 업데이트 버전: `1.3.4`
- 호환 업데이트 채널 버전: `4.0.41`
- 기준 파일: `ErosTower.v1.update.js`

## 스테이터스 패널 반응 속도

- 탭 전환, 인벤토리 페이지 이동, 기술·아이템 선택, 콜드스타트 확인/취소는 이미 열린 패널의 메모리 상태를 바로 사용합니다. 매번 설정·채팅·상태를 다시 읽던 왕복을 제거했습니다.
- 빠른 명령은 `setChatPanel`로 패널 전체를 다시 만들지 않습니다. 기존 surface와 전역 입력 리스너를 유지한 채 `SafeElement.setInnerHTML` 한 번으로 내용부터 바꾸고, 모든 버튼 재탐색·위치 측정은 화면 반영 뒤 백그라운드에서 수행합니다.
- 패널을 그릴 때 계산한 버튼 위치를 실제 탭에 재사용합니다. 정상적인 버튼 입력마다 main DOM 위치를 다시 측정하지 않으며, 캐시에 맞지 않는 입력에서만 스크롤·리사이즈·`details` 레이아웃 변화를 복구하기 위해 다시 측정합니다.
- 포인터·키보드 이벤트 리스너 4개를 병렬로 연결해 패널 재렌더 뒤 입력 대기 시간을 줄였습니다.
- 새로고침, 선택지 재생성처럼 실제 상태 동기화나 모델 호출이 필요한 명령은 버튼을 누른 즉시 진행 중 표시를 먼저 그린 뒤 작업합니다.
- 최근 24회의 전체 마운트·빠른 surface 표시·후행 위치 측정 시간을 구조화해 보존하며, 750ms 이상 걸린 단계는 RisuAI 플러그인 콘솔에 `gameplay panel timing`으로 기록합니다.

## 선택 상태 안전성

- 기술·아이템 타일 선택은 화면을 먼저 갱신하고 저장은 백그라운드에서 수행합니다. 저장 완료를 기다리느라 버튼 표시가 멈추지 않습니다.
- 타일을 누른 직후 Risu 기본 전송 버튼을 눌러도 요청 경계에서 최신 선택 저장을 확인한 뒤 행동을 해석합니다.
- 백그라운드 저장이 실패하면 정확한 최신 선택을 메모리에 보존해 다음 요청 상태에 덮어씌우며, 다른 채팅으로 전환된 오래된 저장은 적용하지 않습니다.
- 빠른 화면 명령도 열린 패널 세션·소유자·scope 일치 검사는 동기적으로 유지합니다. 전체 채팅 재검증은 느린 상태 작업과 최초 마운트에서 계속 수행합니다.

## 굵은 세계관 계열 분류

- 세부 분류를 각각 별도 탭으로 만들지 않습니다. `현장 기술·조사 기술·기계술`은 `기술`, `인술·체술·환술`은 `닌술`, `검술·경공·내공`은 `무공`으로 합칩니다.
- `마법·특성·도술·능력`도 각자의 굵은 계열을 유지합니다. 알려진 굵은 계열로 안전하게 판별할 수 없는 명칭은 새 세부 탭을 만들지 않고 `능력`으로 묶습니다.
- 무기, 보조무기, 마법도구, 옷, 장비, 장신구, 소모품 등 소유물은 모두 `아이템` 한 탭으로 합칩니다.
- 합쳐지는 것은 상단 분류뿐입니다. 각 칸과 상세 정보에는 원래 `categoryLabel`인 `체술`, `검술`, `마법도구` 같은 세부 명칭을 그대로 보존합니다.
- 공격·방어·지원, 주무기·보조무기 같은 역할 태그로 새 분류를 만들지 않습니다.

- 세 배포 파일 모두 `node --check` 통과.
- ESLint 8 정적 검사(`no-redeclare`, `no-dupe-keys`) 통과.
