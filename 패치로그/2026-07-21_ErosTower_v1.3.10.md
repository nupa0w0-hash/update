# Eros Tower 1.3.10 — 으아악씨·모시모씨 RP 응답 경량화

## 증상과 원인

- 으아악씨와 모시모씨는 화면에 짧은 선택지·대사만 내보내지만 요청에는 그보다 훨씬 큰 문맥과 출력 예산이 실렸다.
- 으아악씨는 최대 28,000자 payload와 첫 호출 2,048토큰, 복구 호출 1,536토큰을 사용했다.
- 모시모씨의 에이전트 카드 기본 `contextWindow`는 4였지만 최근 메인 대화 구성 함수가 최소 10턴을 강제하고 전역 설정을 읽어, 에이전트별 설정이 실제 요청에 반영되지 않았다.
- 모시모씨는 최대 9,200자의 검색 근거, 6,800자의 최근 대화, 직접 대화 20개 또는 연락처별 대화 10개를 중복해 실을 수 있었고, 짧은 JSON 답변에도 최대 800토큰을 허용했다.
- 모델·Provider 자체 왕복 시간이 남더라도 이 과한 입력·생성량이 RP 체감 지연을 더 키웠다.

## 변경 사항

### 으아악씨

- 선택지 payload 상한을 28,000자에서 18,000자로 낮췄다.
- 선택지 근거는 최대 10,000자에서 7,000자, 최근 대화는 최대 6개×1,400자에서 4개×1,000자로 줄였다.
- 첫 선택지 호출은 최대 2,048토큰에서 1,024토큰, compact 복구는 1,536토큰에서 768토큰으로 줄였다.
- 기존처럼 모델이 지원하는 가장 낮은 추론 수준을 사용하고, 정확히 3개 선택지·언어·지식 경계·준비된 기술/아이템 검증과 1회 복구는 유지한다.

### 모시모씨

- 에이전트 카드의 `contextWindow`를 실제 최근 메인 대화, 사적 대화, 검색 근거에 공통 적용한다. 기본 4는 더 이상 최소 10으로 확대되지 않는다.
- 기본 4 기준 검색 근거 상한은 9,200자에서 5,600자, 최근 메인 대화 상한은 6,800자에서 3,400자로 줄었다.
- 직접 사적 대화는 최대 20개에서 8개, 능동 연락의 연락처별 대화는 10개에서 4개로 줄였다.
- 중복되는 인물 프로필·관계·대화 필드에 항목별 길이 제한을 적용하고, 직접 통신 payload는 18,000자, 최대 40명 능동 연락 roster payload는 24,000자 상한 안에 맞춘다.
- 짧은 통신 JSON 출력은 최대 800토큰에서 512토큰으로 줄이고 모델이 지원하는 가장 낮은 추론 수준을 사용한다.
- 선택한 Provider·모델·인증 정보는 바꾸지 않으며, 1회 통신 호출·전체 연락처 선택 권한·인물의 답장/무시/지연/거부 판단과 지식 경계는 유지한다.
