# Eros Tower 배포 체크리스트

이 저장소는 RisuAI 업데이트 URL이 직접 바라보는 배포 저장소입니다.

배포할 때는 `ErosTower.v1.update.js`만 갱신하면 안 됩니다. v1 채널과 4.x 호환 채널을 모두 확인해야 합니다.

## 배포 파일

1. `ErosTower.v1.update.js`
   - v1 정식 업데이트 채널.
   - `//@version`은 실제 제품 버전 `1.1.x`를 사용합니다.
   - `//@update-url`은 `https://raw.githubusercontent.com/nupa0w0-hash/update/main/ErosTower.v1.update.js`입니다.

2. `☸에로스 타워.js`
   - 직접 설치용 파일.
   - `ErosTower.v1.update.js`와 동일 내용이어야 합니다.
   - SHA256도 v1 파일과 같아야 정상입니다.

3. `ErosTower.update.js`
   - 구 4.x 설치본을 위한 호환 업데이트 채널.
   - 본문은 최신 v1 패키지와 같아야 합니다.
   - 헤더만 호환 채널용으로 다릅니다.
   - `//@display-name`은 실제 제품 버전 `☸Eros Tower 1.1.x`를 사용합니다.
   - 내부 `const VERSION`도 실제 제품 버전 `1.1.x`를 사용합니다.
   - 단, 비교용 헤더 `//@version`은 `4.0.x`를 계속 증가시킵니다.

## 중요 규칙

- RisuAI는 설치된 플러그인의 `//@update-url`을 따라갑니다.
- 원격 `//@version`이 설치본보다 높을 때만 업데이트가 표시됩니다.
- 예전 4.x 설치본은 `ErosTower.update.js`를 봅니다.
- 따라서 `ErosTower.update.js`의 `//@version`을 `1.1.x`로 내리면 안 됩니다.
- 예: 이전 compat가 `4.0.9`라면 다음 배포 compat는 `4.0.10`이어야 합니다.
- 실제 표시명은 `☸Eros Tower 1.1.51`처럼 제품 버전을 보여주고, 업데이트 비교용 헤더만 `4.0.10`처럼 둡니다.

## 배포 전 검증

PowerShell에서 이 저장소 루트 기준으로 실행합니다.

```powershell
node --check .\ErosTower.v1.update.js
node --check .\ErosTower.update.js
node --check ".\☸에로스 타워.js"
git diff --check
Select-String -Path .\ErosTower.v1.update.js,.\ErosTower.update.js,".\☸에로스 타워.js" -Pattern '^//@(display-name|version|update-url)'
```

호환 채널 본문 동기화 확인:

```powershell
$a = Get-Content -Encoding UTF8 ".\ErosTower.v1.update.js"
$b = Get-Content -Encoding UTF8 ".\ErosTower.update.js"
$bodyA = $a | Select-Object -Skip 5
$bodyB = $b | Select-Object -Skip 5
Compare-Object $bodyA $bodyB
```

정상이라면 비교 결과가 비어 있어야 합니다. 상단 5줄 헤더만 다르고 본문은 같아야 합니다.

## 배포 후 확인

푸시 후 아래 두 raw URL을 모두 확인합니다.

```text
https://raw.githubusercontent.com/nupa0w0-hash/update/main/ErosTower.v1.update.js
https://raw.githubusercontent.com/nupa0w0-hash/update/main/ErosTower.update.js
```

완료 조건:

- v1 raw URL이 최신 `//@version 1.1.x`를 반환해야 합니다.
- compat raw URL이 최신 `//@version 4.0.x`를 반환해야 합니다.
- 두 raw URL의 `//@display-name`은 같은 제품 버전이어야 합니다.

GitHub raw branch URL은 CDN 캐시가 있어 몇 분 동안 이전 파일을 반환할 수 있습니다. 두 URL이 모두 최신 헤더를 반환하기 전에는 배포 확인 완료로 보지 않습니다.

## 패치로그

배포마다 `패치로그/YYYY-MM-DD_ErosTower_vX.Y.Z.md`에 아래 항목을 기록합니다.

- 변경 버전과 날짜
- 문제 증상과 원인
- 수정한 파일
- 백업 위치
- 검증 명령과 결과
- v1 raw URL 확인 결과
- compat raw URL 확인 결과
